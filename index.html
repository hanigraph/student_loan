Student Loan Repayment Calculator

Introduction

FastAPI

Purpose: FastAPI is a modern, fast (high-performance) web framework for building APIs with Python 3.7+ based on standard Python type hints. It is designed to create RESTful APIs quickly and efficiently.

Main Functions:

  • Build APIs using Python type hints and automatic data validation.
  • Generate interactive API documentation (Swagger UI and ReDoc).
  • Handle asynchronous requests to improve performance.

Frontend (Static Site)

Purpose: A static site that provides a user interface for interacting with the FastAPI backend. The frontend collects input from users and displays the calculated loan repayment results.

Main Functions:

  • Collect user inputs for loan calculation.
  • Send data to the FastAPI backend.
  • Display results and handle user interactions.

The Test Plan

Inputs to the FastAPI Backend:

  • age: Integer, represents the age of the applicant.
  • loan_amount: Decimal, represents the total amount of the loan.
  • years_to_repay: Integer, represents the number of years over which the loan will be repaid.
  • interest_rate: Decimal, represents the annual interest rate of the loan.

Acceptable and Unacceptable Values

age:

  • Acceptable: Integer values between 18 and 30 (inclusive).
  • Not Acceptable: Values less than 18, greater than 30, non-integer values.

loan_amount:

  • Acceptable: Decimal values between 1000 and 500000 (inclusive).
  • Not Acceptable: Values less than 1000, greater than 500000, negative values, non-numeric values.

years_to_repay:

  • Acceptable: Integer values between 1 and 20 (inclusive).
  • Not Acceptable: Values less than 1, greater than 20, non-integer values.

interest_rate:

  • Acceptable: Decimal values between 0 and 10 (inclusive).
  • Not Acceptable: Values less than 0, greater than 10, negative values, non-numeric values.

Conclusion

In this project, we developed a loan repayment calculator using FastAPI for the backend and a static frontend for user interaction. The FastAPI framework was chosen for its speed, modern features, and ease of use in creating RESTful APIs. The frontend allows users to input loan details and receive calculated monthly payments.

Testing was crucial to ensure that the API correctly handles a variety of inputs, including edge cases and invalid data. The test cases cover a range of scenarios to ensure robustness and validate that the application correctly processes valid inputs while handling errors gracefully.

By following this test plan and analyzing the results, we ensure that the application is reliable and meets the specified requirements, providing a seamless user experience for loan calculations.
